Session 2
What We're Building
Multi-Project Data Structure First
S.projects[] array replaces single S.rooms/S.proj. Each project has its own client, rooms, invoices, quote, timeline. Migration wraps existing data into projects[0] automatically — nothing lost.
Project Board Dashboard First
Replaces the single-project dashboard. Cards show: client name, address, project status (Estimating / Active / Invoiced / Complete), total value, days since last activity.
New Project Wizard First
3-step flow: (1) Pick or create client, (2) Job type intake, (3) Project name + address + start date. Skip and back always available. Drops you into rooms when done.
Job Type Intake Important
Full Reno, Partial Reno, Service Call, Repair, New Build. Determines which tools show up. Full Reno = room calculator + all trades. Repair = line items only. Think about what questions to ask before the session.
Trade Toggles Important
Enable/disable each trade per room. Disabled trades disappear from quote, timeline, shopping list everywhere. Pre-selected by room type — bathroom shows 6 relevant trades, not 12.
Morning Briefing — All Projects Important
Now pulls from all projects. Overdue invoices across all jobs, deposits pending, projects with no activity 7+ days, projects behind schedule. Clean state when nothing needs attention.
+ More Options on Trades If time
Less common line items hidden behind a "+ More" button at the bottom of each trade. Reduces overwhelm for simple jobs, full depth still available for complex ones.
